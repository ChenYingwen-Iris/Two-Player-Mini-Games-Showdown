## 🎮 功能特性

### 轮盘系统
- **四宫格布局**: 四个游戏平均分布在轮盘上
- **颜色标识**: 
  - 数蝴蝶: 粉红色 (255, 150, 200)
  - 双人迷宫: 青绿色 (150, 200, 100)
  - 硬币收集: 金黄色 (255, 200, 100)
  - 拔河大战: 紫色 (200, 150, 255)
- **动画效果**: 减速旋转，自然停止
- **智能选择**: 只从未玩过的游戏中随机选择

### 积分系统
- 玩家1 vs 玩家2
- 每局获胜 +5 分
- 实时分数显示
- 最终排名展示

### 用户界面
- **主菜单**: 显示轮盘和当前分数
- **游戏选择**: 旋转动画 + 选中提示
- **手动输入**: 游戏结束后输入胜者
- **最终界面**: 显示总分和最终赢家

## 📝 使用流程

### 完整游戏流程

```
1. 启动游戏合集
   └─> python game_launcher.py
       或
   └─> ./start.sh

2. 主菜单界面
   └─> 显示轮盘（四个游戏）
   └─> 显示当前分数 (初始 0:0)
   └─> 按 SPACE 开始旋转

3. 轮盘旋转
   └─> 自动旋转并减速
   └─> 随机选中一个未玩过的游戏
   └─> 显示选中的游戏名称

4. 开始游戏
   └─> 按 ENTER 确认
   └─> 在新终端运行对应游戏
   └─> 玩家进行游戏

5. 输入结果
   └─> 游戏结束后返回启动器
   └─> 弹出输入界面
   └─> 按 1/2/0 选择胜者

6. 更新状态
   └─> 胜者获得 5 分
   └─> 该游戏标记为已玩（变灰）
   └─> 返回主菜单

7. 重复步骤 2-6
   └─> 直到四个游戏全部完成

8. 最终结果
   └─> 显示双方总分
   └─> 显示最终获胜者
   └─> 按 ESC 退出
```

## 💡 设计决策

### 为什么使用手动输入胜者？

由于各个子游戏是独立开发的，直接集成存在以下挑战：
1. **独立的 pygame 实例**: 每个游戏都初始化自己的窗口和事件循环
2. **不同的返回机制**: 有些游戏没有返回值设计
3. **复杂的状态管理**: 需要大幅修改原游戏代码

**解决方案**: 
- 使用手动输入方式，保持子游戏的独立性
- 启动器负责管理轮盘、积分和流程
- 简单、可靠、易于维护

## 🔧 技术栈

- **Python 3.7+**

- **Math & Random**: 轮盘旋转算法
